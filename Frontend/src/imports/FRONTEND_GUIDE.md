# IntelliJob - Frontend Development Guide

Hello! This document outlines the backend architecture we've built for the **Autonomous AI Career Agent (IntelliJob)** and provides the exact requirements and API contracts you need to connect your React frontend.

## 📱 Required UI Sections & Functionality

Based on the backend workflow, your frontend should include the following core sections:

1. **Dashboard / Mission Control**
   * **Purpose**: High-level overview of active workflows.
   * **Data Provided**: Overall stats, active LangGraph status (running/idle), and recent system notifications.

2. **Job Discovery Board**
   * **Purpose**: Display jobs fetched and ranked by the AI.
   * **Data Provided**: List of jobs including `title`, `company`, `location`, `description`, `apply_url`, and the AI-generated `score` and `reason` for the match.

3. **Application Tracker**
   * **Purpose**: Track the status of job applications.
   * **Data Provided**: List of applications with their `status` (Pending, Applied, Interviewing) and timestamps.

4. **Interview Prep Hub**
   * **Purpose**: View generated interview questions for specific roles.
   * **Data Provided**: Array of specific technical and behavioral questions generated by the AI for a target job.

5. **Settings / Preferences**
   * **Purpose**: Form for the user to input their core preferences.
   * **Data Provided/Expected**: User preferences like Startup vs. MNC, domains of interest, and remote/hybrid preference.

## 🔌 API Contract (FastAPI)

The backend runs locally on `http://127.0.0.1:8000`. All routes are prefixed with `/api`.

### 1. Health Check
* **Endpoint**: `GET /api/health`
* **Response**: `{"status": "healthy"}`
* **Usage**: Ping this on app load to ensure the backend is running.

### 2. Trigger the AI Workflow
* **Endpoint**: `POST /api/workflow/trigger`
* **Payload**: 
  ```json
  {
    "user_id": 1
  }
  ```
* **Response**: `{"status": "Workflow triggered asynchronously"}`
* **Usage**: This triggers the multi-agent system to fetch jobs, rank them, tailor resumes, and simulate applications in the background.

### 3. Fetch Discovered Jobs
* **Endpoint**: `GET /api/jobs`
* **Response**:
  ```json
  {
    "jobs": [
      {
        "id": 1,
        "title": "Backend Software Engineer",
        "company": "Tech Startup AI",
        "location": "Remote",
        "description": "...",
        "apply_url": "https://...",
        "source": "Mock API"
      }
    ]
  }
  ```
* **Usage**: Display these on the Job Discovery Board.

### 4. Fetch User Applications
* **Endpoint**: `GET /api/applications`
* **Response**:
  ```json
  {
    "applications": [
      {
        "id": 1,
        "job_id": 12,
        "status": "Pending",
        "applied_at": "2026-05-08T10:00:00Z"
      }
    ]
  }
  ```
* **Usage**: Populate the Application Tracker.

### 5. Fetch Notifications / Activity Log
* **Endpoint**: `GET /api/notifications`
* **Response**:
  ```json
  {
    "notifications": [
      {
        "id": 1,
        "message": "Notification agent sent updates via Telegram mock.",
        "type": "System",
        "sent_at": "2026-05-08T10:00:00Z"
      }
    ]
  }
  ```
* **Usage**: Show a dropdown or sidebar activity feed of what the agents are doing.

### 6. On-Demand Resume Optimization
* **Endpoint**: `POST /api/resume/optimize?user_id=1`
* **Response**: `{"status": "success", "message": "Resume tailored successfully (mock)."}`
* **Usage**: Triggered when a user manually asks the AI to generate a LaTeX resume variant for a job.

### 7. Trigger Job Application (Playwright)
* **Endpoint**: `POST /api/apply?job_id=1`
* **Response**: `{"status": "success", "message": "Application workflow started..."}`
* **Usage**: Called when the user clicks "Auto Apply" on a job card.

### 8. Generate Interview Questions
* **Endpoint**: `POST /api/interview/questions?job_id=1`
* **Response**:
  ```json
  {
    "questions": [
      "What are your core strengths?",
      "How do you build APIs?"
    ]
  }
  ```
* **Usage**: When a user clicks "Prepare for Interview" on a job they applied to.
